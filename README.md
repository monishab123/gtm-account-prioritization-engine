# Account Prioritization + AI Outreach Engine

**What this is:** a working prototype of a GTM automation pipeline — it takes
a list of target accounts, scores and auto-tiers them (1:1 / 1:few / 1:many),
and drafts personalized outreach for the accounts worth a real human's time.

**The problem it solves:** sales and marketing teams can't manually review
every account in a pipeline to decide who gets a hand-written email, who gets
a lighter-touch campaign, and who gets generic nurture. That triage is
usually done by eyeballing a spreadsheet. This automates the sorting and
writes the first draft of the outreach for the top tier — turning hours of
manual account review into a live, adjustable, instant view.

**Try it:** open `app.html` in any browser. Drag the scoring-weight sliders
and watch every account re-tier live. Click a row to see why it scored the
way it did, and to read/edit the AI-drafted outreach note. Search, filter,
sort, and export the current view to CSV.

---

## Files

- **`app.html`** — the interactive product. Open it directly in a browser.
- **`accounts.csv`** / **`score_and_draft.py`** — a command-line version of
  the same scoring + drafting logic, for batch runs (e.g., a nightly job
  against a CRM export) instead of the interactive browser tool.
- **`scored_accounts.csv`** / **`outreach_drafts.json`** — example output
  from the CLI version.

## Architecture

```
CRM export (CSV) → scoring engine (engagement, ICP fit, recency, tech signal)
                 → tiering (1:1 / 1:few / 1:many)
                 → outreach drafting (top tier only)
                 → export / CRM push
```

Scoring weights are configurable (sliders in the app, `WEIGHTS` dict in the
Python script) — in production these get tuned against real win-rate data
instead of guessed defaults.

## What's real vs. stubbed

- **Real:** the scoring math, weighting, tiering logic, filtering/sorting,
  and CSV export all genuinely run — nothing is hardcoded per account.
- **Stubbed for the demo:** `accounts.csv` is synthetic data, and outreach
  copy is templated rather than generated by a live LLM call, so the whole
  thing runs with zero API keys or CRM credentials. Production version would:
  (1) point ingestion at a real HubSpot/Salesforce export or API, (2) replace
  the template in `draftFor()` / `draft_outreach()` with a live LLM call
  (Claude/GPT) using the account + reasons as context, (3) add auth and a
  real CRM write-back instead of a CSV download.

## Why this exists

This mirrors the GTM Engineer skill set companies are hiring for: SQL/data
fluency, scoring model design, and LLM-powered orchestration — built on top
of 1:1/1:few/1:many ABM tiering logic drawn from hands-on account-based
marketing work, not a tutorial.
